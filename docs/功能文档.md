# AI 面试助手 — 功能文档

> 版本 0.1.0 | 技术栈：Next.js 15 + Prisma + MySQL + Tailwind CSS

---

## 一、项目概述

AI 面试助手是一个面向技术岗位面试准备的全栈 Web 应用。核心能力包括：

- **智能刷题**：自建题库 + AI 点评打分
- **模拟面试**：AI 扮演面试官，逐轮提问并追问，支持语音对话
- **题库广场**：跨用户浏览公开题目，按方向/难度筛选
- **分享码同步**：用户间通过 8 位分享码拉取公开题库和试卷
- **试卷系统**：组卷、限时考试、AI 评分
- **管理后台**：用户管理、AI 模型配置、Prompt 管理

## 二、用户体系

### 2.1 角色

| 角色 | 说明 |
|------|------|
| 游客 | 可浏览首页和公开主页，不可使用核心功能 |
| 普通用户 | 注册后默认角色，可创建题库、刷题、面试、组卷、同步他人题库 |
| 管理员 | 额外拥有管理后台权限：用户管理、模型配置、Prompt 管理、题目下架 |

### 2.2 认证

- 基于 JWT（jose），Token 存于 httpOnly Cookie，7 天有效
- 密码使用 bcryptjs 加盐哈希存储
- 中间件保护所有核心路由（`/practice`、`/paper`、`/interview`、`/history`、`/explore`、`/admin`）
- 权限相关字段（`role`、`canCreate`）不写进 JWT，每次按需从 DB 读取，保证管理员禁权后立即生效

### 2.3 分享码

- 每位用户可生成唯一的 8 位分享码（32 字符集，排除易混淆字符 0/O/1/I/L）
- 通过分享码，其他用户可访问公开主页（`/u/<code>`），浏览其公开的题库、试卷
- 重置冷却时间：24 小时

## 三、核心功能模块

### 3.1 题库管理（/practice）

**话题树**
- 每个用户拥有一棵私有话题树，根节点为 null，支持无限嵌套
- 支持 CRUD、拖拽排序、批量移动
- 可见性控制：`private`（仅自己）、`public`（对码可见）、`force_off`（强制隐藏，子节点全部不可见）

**题目 CRUD**
- 每道题包含：标题、题干（富文本，Tiptap 编辑器）、方向、难度、标签、参考答案、详细答案
- 方向（Direction）：前端 / 后端 / 客户端 / 算法 / 测试 / 产品 / 其他
- 难度：简单 / 中等 / 困难
- 支持软删除（`deletedAt`），误删可恢复

**题目导入**
- 支持粘贴文本批量导入（AI 自动提取题目信息）
- 支持 JSON 格式批量导入
- 导入流水线：文本 → AI 分类（识别方向/难度）→ AI 提取（标题/题干/答案）→ 批量写入

**练习模式**
- 单题练习：查看题目 → 写下答案 → AI 点评打分
- 记录保存：每次练习自动保存，支持回看历史记录和 AI 反馈

### 3.2 试卷系统（/paper）

**试卷管理**
- 从题库中选题组卷，支持排序、限时设置
- 可公开试卷（广场可见）

**考试流程**
1. 开始作答：计时器启动（可选限时）
2. 逐题作答：每道题可写答案
3. 提交批改：AI 逐题评分并给出反馈
4. 成绩单：总分 + 每题得分 + AI 评语

**作答记录**
- 每次作答自动保存，支持回看历史成绩和反馈

### 3.3 题库广场（/explore）

- 浏览所有用户公开的题目
- 按方向（前端/后端/算法等）和难度筛选
- 热度排序：`manualHeat × 100 + cloneCount × 10 + likeCount × 5 + viewCount × 1`
- 支持点赞、查看详情、一键克隆到自己题库

### 3.4 模拟面试（/interview）

**面试流程**
1. 选择目标岗位（前端/后端/全栈/算法/测试/产品）和级别（初级/中级/高级）
2. AI 面试官逐轮提问，支持顺着回答追问
3. 面试结束后 AI 自动生成综合评分和总结

**语音对话**
- 双模式语音输入：
  - **native**：浏览器 Web Speech API，免费、实时转写
  - **server**：录音上传后端 ASR（Whisper 兼容 API），高精度
- 语音输出（TTS）：浏览器 SpeechSynthesis，AI 回复可朗读
- 默认优先实时识别（native），不可用时自动退到 server 模式

### 3.5 分享码同步（Navbar 同步按钮）

**同步方式**
- **按目录同步**：选择对方公开话题树中的特定分支，拉取题目
- **按试卷同步**：选择对方公开试卷，拉取试卷及关联题目
- **一键全量同步**：拉取对方所有公开目录和试卷

**同步机制**
- 目录树深度优先重建，保持父子结构
- 题目按标题去重（同用户下不重复创建）
- 同步记录可追溯（状态、新建/跳过数量、时间）

**公开主页（/u/<code>）**
- 展示用户公开的题库统计、话题树、题目列表、试卷列表
- 访客可直接同步感兴趣的内容

### 3.6 管理后台（/admin）

**用户管理**
- 查看所有用户列表
- 修改角色（普通用户 / 管理员）
- 控制加题权限（`canCreate`，关闭后用户无法新建题目）

**AI 模型配置**
- 支持四种角色模型：主模型（primary）、答案模型（answer）、视觉模型（vision）、语音识别（asr）
- 每类可配置多个模型（来源：数据库配置 / 环境变量兜底），按优先级链式调用
- 支持 OpenAI 兼容 API（DeepSeek、通义千问、智谱 GLM 等）
- 自动故障转移：当前模型失败时尝试下一个
- 记录最后一次错误，便于排查

**Prompt 管理**
- 支持自定义各类 AI Prompt（面试官系统提示、答题评估、答案生成等）
- 变量插值（`{{role}}`、`{{level}}` 等）
- 支持预览和重置为默认值

**题目管理**
- 查看全平台题目，支持下架操作（`isDelisted`）

## 四、技术架构

### 4.1 前端

| 层级 | 技术选型 |
|------|---------|
| 框架 | Next.js 15（App Router） |
| UI | React 19 + Tailwind CSS 3 |
| 富文本 | Tiptap（基于 ProseMirror） |
| 语音输入 | Web Speech API + MediaRecorder |
| 语音输出 | Web SpeechSynthesis |
| 图标 | 自定义 SVG Icon 组件（Lucide 风格） |

### 4.2 后端

| 层级 | 技术选型 |
|------|---------|
| API | Next.js Route Handlers |
| 认证 | JWT（jose）+ bcryptjs |
| ORM | Prisma 6 |
| 数据库 | MySQL（开发环境 127.0.0.1:3306） |
| AI 接口 | OpenAI 兼容 `/chat/completions` + `/audio/transcriptions` |

### 4.3 数据模型（核心）

| 模型 | 说明 |
|------|------|
| User | 用户（含角色、权限、分享码） |
| Topic | 话题树节点（嵌套、可见性控制） |
| Question | 题目（含热度分量、可见性、软删除） |
| PracticeRecord | 练习记录 |
| TestPaper + TestPaperItem | 试卷与题目关联 |
| TestPaperAttempt | 考试作答记录 |
| InterviewSession + InterviewMessage | 面试会话与消息 |
| AiModelConfig | AI 模型配置（多模型、故障转移） |
| AiPromptConfig | AI Prompt 模板 |
| SyncRecord | 同步记录 |

## 五、环境要求与启动

### 5.1 环境变量

```env
DATABASE_URL=mysql://root:root@127.0.0.1:3306/interview_agent
DEEPSEEK_API_KEY=sk-xxx      # 主模型 API Key（必填）
DEEPSEEK_BASE_URL=https://api.deepseek.com
DEEPSEEK_MODEL=deepseek-chat
# 以下可选（不填则复用主模型配置）
ANSWER_API_KEY=
ANSWER_BASE_URL=
ANSWER_MODEL=
VISION_API_KEY=
VISION_BASE_URL=
VISION_MODEL=
ASR_API_KEY=
ASR_BASE_URL=
ASR_MODEL=
```

### 5.2 启动

```bash
npm install          # 安装依赖 + prisma generate
npx prisma db push   # 同步数据库结构
npm run db:seed      # 可选：填充种子数据
npm run dev          # 启动开发服务器（Turbo 模式）
```

### 5.3 设置管理员

```bash
npm run set-admin    # 将第一个用户设为管理员
```
